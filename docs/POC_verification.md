# POC（P0/P1）動作確認ガイド

POCの動作確認は3段階で用意しています。

## 前提：依存インストール（初回のみ）

Chromium 等も含めてインストールされます。

```bash
cd toiawase_form_sales_auto_system
npm install
```

---

## ① 一番簡単：自動E2E（外部ネットワーク不要・数十秒）

```bash
npm run e2e
```

ローカルに立てたテストフォーム（**確認画面あり2段／1段／ハニーポット付き**）に対して、
発見→解析→文面生成→Plan→Execute→成功判定 を自動で通し、各項目を PASS/FAIL 表示します。
最後に **`ALL PASSED`** が出れば P0 の一連（§12のPoCゴール）が成立しています。

確認できること：

- フィールドマッピング（会社名/氏名/メール/本文）
- **ハニーポットを検知して埋めない**（埋めるとサーバーが弾くので、成功＝スキップできた証明）
- 確認画面のハンドリング（Planでスクショ、Executeで確認→送信）
- 成功判定（L5）

---

## ② ブラウザを目で見ながら確認

**A. E2Eをヘッドレス解除で実行**（ブラウザが実際に入力していく様子が見えます）

```bash
HEADLESS=false npm run e2e
```

**B. テストフォーム自体を手で触る**

```bash
npm run testform      # → http://127.0.0.1:8787/contact が起動
```

ブラウザで開くと、システムが相手にする「確認画面あり2段フォーム」の実物を確認できます。

---

## ③ 半自動フルフロー（P1の体験・承認ダッシュボード込み）

実際の運用の流れをローカルで一周します。

```bash
# 1. リストを取り込む（L0）※サンプル同梱
npm run cli -- ingest scripts/sample_companies.csv

# 2. デモの承認待ちを1件投入（実サイトを叩かず確認用）
npx tsx scripts/seed_pending.ts

# 3. 承認ダッシュボードを起動
npm run serve         # → http://localhost:4599
```

ブラウザで `http://localhost:4599` を開くと、承認待ち企業がスクショ・文面・gate付きで並びます。
そこで **承認 / 却下 / 抑制 / 送信実行** を操作できます（これが半自動の「人は最終承認だけ」の画面）。

> ⚠️ ②③で `execute`（実送信）まで行くのは**ローカルのテストフォームだけ**にしてください。
> `.env` を設定して実企業へ送るのは、`SENDER_*`（送信元情報）を正しく入れてからにしましょう（コンプラ§9）。

---

## （おまけ）実サイトで「発見」だけ試す

送信はせず、L1のフォーム発見だけを実ドメインで確認できます：

```bash
npm run discover-batch -- smarthr.jp cybozu.co.jp <相手ドメイン>
```

発見方法（common_path / browser_render 等）・信頼度・フォームURL・発見率が表として出ます。

---

## まとめ

まず **`npm run e2e`** を実行して `ALL PASSED` を確認するのが、POC成立の最短確認です。
エラーが出た場合はその出力を確認してください。
